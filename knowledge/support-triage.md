### Когда нужны данные пользователя, а когда нет

Republic (republicmag.io) — медиа с подпиской. Подписка покупается на Redefine (redefine.media). Аккаунты на двух сайтах независимы.

#### Нужны данные (account_info, subscription_info, payments_info, audit_log):

- «За что списали деньги?», «Что за списание?» → payments_info, subscription_info, account_info
- «У меня не работает подписка», «Не могу читать» → subscription_info, account_info
- «Хочу отменить подписку», «Верните деньги» → subscription_info, payments_info
- «У меня два аккаунта», «Подписка на другой почте» → account_info, subscription_info
- «Не могу войти», «Нет аккаунта» → account_info
- «Меня заблокировали» → account_info, audit_log
- Любая проблема с конкретным пользователем, где решение зависит от состояния аккаунта/подписки

#### НЕ нужны данные:

- «Как купить подписку?», «Как оплатить в рублях?» — общий вопрос, ответ есть в базе знаний
- «Забыл пароль» — общая инструкция, не зависит от данных
- Предложения статей, обращения в редакцию
- Уведомления от сервисов, спам
- Неясное обращение типа «подписка» без контекста — сначала уточнить, потом запрашивать данные
- Вопросы, на которые можно ответить без персональных данных
