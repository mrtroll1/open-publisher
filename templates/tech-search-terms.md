Определи, нужен ли доступ к кодовой базе для ответа на этот запрос. Если нужен — извлеки поисковые термины.

Ищи: названия фич, компонентов, страниц, шаблонов, API-эндпоинтов, технические термины.

## Запрос
{{EMAIL}}

Верни JSON: {"search_terms": ["term1", "term2", ...], "needs_code": true/false}

- search_terms — до 5 ключевых терминов для grep по коду. Если needs_code = false, верни пустой список [].
- needs_code — нужен ли поиск по кодовой базе для ответа.
  - needs_code = true если ответ требует знания кода, конфигурации или реализации: баги, фичи, вёрстка, метатеги, шаблоны, настройки, API, интеграции, как что-то работает или как что-то поменять.
  - needs_code = false только для чисто организационных/процедурных вопросов, не связанных с реализацией: подписка, оплата, возврат, пароль, доступ к контенту.
  Примеры:
  - «Можно превью для ссылок в соцсетях сделать другим?» → needs_code: true (OG-метатеги, шаблоны)
  - «Как поменять описание журнала на сайте?» → needs_code: true (шаблон/CMS)
  - «Страница подписки не загружается, белый экран» → needs_code: true (баг)
  - «При нажатии на кнопку "Отменить" ошибка 500» → needs_code: true (баг)
  - «Не могу отменить подписку» → needs_code: false (процедура)
  - «Списали деньги дважды» → needs_code: false (оплата)
