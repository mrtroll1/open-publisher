Извлеки поисковые термины из письма в техподдержку для поиска по кодовой базе.

Ищи: названия ошибок, пути, имена функций/компонентов, технические термины, названия фич.

## Письмо
{{EMAIL}}

Верни JSON: {"search_terms": ["term1", "term2", ...], "needs_code": true/false}

- search_terms — до 5 ключевых терминов для grep по коду
- needs_code — нужен ли поиск по кодовой базе. По умолчанию false.
  - needs_code = false для пользовательских вопросов: подписка, оплата, возврат, аккаунт, пароль, доступ к контенту.
  - needs_code = true только для технических проблем, которые могут быть вызваны багом в коде: ошибки на странице, неработающие кнопки, сломанная верстка, 500/404, проблемы с API.
  Примеры:
  - «Не могу отменить подписку» → needs_code: false (вопрос о процедуре)
  - «При нажатии на кнопку "Отменить" страница выдаёт ошибку 500» → needs_code: true (возможный баг)
  - «Списали деньги дважды» → needs_code: false (вопрос об оплате)
  - «Страница подписки не загружается, белый экран» → needs_code: true (возможный баг)
